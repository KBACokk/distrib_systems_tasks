# Вопросы для самопроверки

1. В чем разница между Forward Proxy (обычный прокси) и Reverse Proxy (обратный прокси)?

Forward Proxy обычно используется клиентом, чтобы выйти во внешнюю сеть (скрывая клиента).

Reverse Proxy (обратный прокси) стоит перед вашими сервисами и защищает их. В этом случае клиент общается с прокси, а тот «под капотом» пересылает запросы во внутреннюю сеть (скрывая серверы бэкенда)

2. Что такое Upstream в терминологии Nginx?

Это «настоящий» сервис (бэкенд), куда прокси-сервер пересылает входящий запрос после обработки правил маршрутизации.

3. Зачем скрывать реальные IP-адреса сервисов от клиента?

Это необходимо для безопасности: можно закрыть прямые порты микросервисов фаерволом, оставив доступ только через Gateway. Также это позволяет использовать единый домен для всех микросервисов вместо множества отдельных IP-адресов.

4. Какие еще задачи, кроме маршрутизации, может выполнять API Gateway (назовите хотя бы 2)?

SSL/TLS терминация: настройка сертификатов безопасности только на шлюзе.
Балансировка нагрузки: распределение запросов между несколькими копиями одного и того же сервиса для повышения стабильности.

5. Что произойдет, если Gateway упадет? Как можно от этого защититься?

Если Gateway упадет, клиент не сможет получить доступ к микросервисам, образуется точка отказа. Можно развернуть несколько шлюзов.
