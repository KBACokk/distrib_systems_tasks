# Вопросы для самопроверки

1. Зачем нужен Signaling сервер, если WebRTC — это P2P технология?

Он нужен лоя координации взаимодействия между клиентами. Он позволяет клиентам обмениваться информацией, необходимой для установления соединения.

2. Что такое NAT и почему он мешает прямому соединению?

NAT (Network Address Translation) - технология, которая позволяет несколькоим устройствам в локальной стеи, использовать один внешний ip для всеимодейсвия с другими устройствами в глобаольной сети.

3. В чем разница между STUN и TURN серверами?

STUN помогает устройствам обнаружить сторонние стройства для установления соединения, в то время как TURN передаёт данные через себя, если они не могут соедениться напрямую.

4. Что такое ICE Candidate?

ICE Candidate (Interactive Connectivity Establishment) - это идентификатор способа связи, который используется бразузером для установления соединения.

5. Почему WebRTC обычно использует UDP, а не TCP?

использование UDP протокола в реальных условиях позволяет добиться минимальных задержек

6. Что такое SDP (Session Description Protocol) и какая информация в нем содержится?

SPD (Session Description Protocol) - это сетеой проткол прикладного уровня, предназначенный для описания сессии передачи потоковых данных. Он сообщает устройствам участников о технических параметрах сессии: адресах, портах, форматах медиа, времени начала и окончания сессии.
